# Today Dashboard

## Overview

The Today view (`Features/Main/Views/TodayView.swift`) is the app's primary hub. It displays the current class schedule, countdowns, school day info, and quick links. Its appearance adapts dynamically based on authentication state, day of week, holiday mode, and current class activity.

## View Hierarchy

```
TodayView
  ├── TodayHeaderView (greeting, date, override/holiday indicators)
  ├── TodayMainContentView
  │     ├── [Not authenticated] → SignInPromptCard
  │     ├── [Holiday mode] → HolidayModeCard
  │     ├── [Loading] → UpcomingClassSkeletonView
  │     ├── [Weekend] → WeekendCard
  │     └── [Weekday with classes]
  │           ├── UnifiedScheduleCard (full day timeline with live progress)
  │           └── QuickLinksCard (2x2 navigation grid)
  └── ScheduleSettingsSheet (modal: day override, holiday toggle)
```

## State Sources

| Source | Data |
|--------|------|
| `ClasstableViewModel` (env object) | Timetable, selected day, upcoming class, holiday mode |
| `AuthServiceV2.shared` | Authentication state, user info |
| `GradientManager` (env object) | Background gradient colors/animation |
| `Configuration` (UserDefaults) | Display preferences, holiday settings |
| `ClassPeriodsManager` | Period times, current/next period |

## Day Override System

Users can preview any weekday's schedule without changing the actual date:

- **Selected Day Override** -- Picks Mon-Fri from `ScheduleSettingsSheet`
- **Set As Today** -- Treats the selected day as if it were today (shows countdowns, progress bars)
- **Holiday Mode** -- Overrides all schedule display with holiday card; optional end date

Day overrides auto-reset when the calendar date changes (checked on `onAppear`).

## Gradient Context

`TodayView.updateGradientColors()` determines the background gradient based on current state:

| State | Gradient |
|-------|----------|
| Not signed in | `notSignedIn` preset |
| Holiday mode | `holiday` preset |
| Weekend | `weekend` preset |
| In active class | Subject-specific colors derived from class name |
| Upcoming class | Lighter subject-specific colors |
| In self-study | `selfStudy` preset |
| Upcoming self-study | `upcomingSelfStudy` preset |
| After school | `afterSchool` preset |
| Default | Normal app gradient |

Subject-specific gradients are generated by parsing the class cell data, extracting the subject name, getting the base subject color from `ModernScheduleRow.subjectColor(for:)`, and creating a 4-color gradient with white, lighter, and darker variants.

## Cards

### UnifiedScheduleCard

The most complex card. Displays every period for the selected day with real-time progress tracking.

- Uses `TimelineView(.periodic(from: .now, by: 1))` for 1-second updates
- Shows countdown timer ("ends in M:SS") for active periods
- Progress bar fills during active periods
- Past periods shown with reduced opacity
- Supports "Set As Today" effective date mode
- Header gradient matches first non-self-study subject color

### Context-Specific Cards

| Card | When Shown | Visual |
|------|------------|--------|
| `NoClassCard` | All classes finished | Green/mint gradient (cheerful) or indigo (evening) |
| `WeekendCard` | Saturday/Sunday | Yellow-to-orange with breathe animation |
| `HolidayModeCard` | Holiday mode enabled | Orange-to-red with optional end date |
| `SignInPromptCard` | Not authenticated | Sparkles icon with button to settings |
| `SchoolDaySummaryCard` | School day | Assembly, arrival, lunch times |
| `DailyScheduleCard` | School day | Full period list with expand/collapse |
| `QuickLinksCard` | Always (authenticated) | 2x2 navigation grid |
| `ClassSummaryCard` | Active/upcoming class | Subject, teacher, room, time badge |

### Assembly/Arrival Times

Hardcoded per weekday:
- **Monday**: Assembly 7:45, Arrival 7:40
- **Tuesday-Friday**: Arrival 7:55
- **Weekend**: N/A

## Live Activity Integration

TodayView coordinates with `ClassActivityManager`:

1. On timetable load → `ClassActivityManager.setTimetable()`
2. On holiday mode change → `ClassActivityManager.setHolidayActive()`
3. Starts Live Activity when classes exist (30 min before first class)
4. Timer-based class transition detection every 10 seconds

## Skeleton Loading

`UpcomingClassSkeletonView` shows gray rectangles with `.shimmering()` animation while timetable data loads, preventing layout jumps.

## Animation System

- Cards use `staggeredEntry(index:animate:)` for cascading appearance (0.08s delay per index)
- Content transitions triggered by `contentStateID` changes (e.g., "schedule-2" for Tuesday)
- Spring animations with `bouncy` response curves
